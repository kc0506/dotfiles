# Initialize Vite + React + shadcn/ui Project

Scaffold a new production-ready Vite + React + TypeScript + shadcn/ui project with linting, formatting, and git hooks.

**IMPORTANT: Follow these steps EXACTLY. Do NOT manually install packages or edit config files beyond what is specified. Use CLI commands as the primary setup method.**

## Pinned Versions (verified 2026-03-09)

| Package | Version | Notes |
|---------|---------|-------|
| shadcn | 4.0.2 | CLI for scaffolding |
| vite | 7.2+ | Generated by shadcn |
| react | 19.2+ | Generated by shadcn |
| typescript | 5.9+ | Generated by shadcn |
| tailwindcss | 4.1+ | v4, CSS-native, no config file |
| oxlint | 1.51+ | Stable since v1.0 (Aug 2025) |
| oxfmt | 0.36+ | Beta (Feb 2026), adopted by Vue/Turborepo/Sentry |

> **Upgrade note:** When bumping shadcn version, re-run the full flow on a test project first. If the generated `eslint.config.js` or `package.json` structure changes, update the `sed`/config patching commands below accordingly.

## Arguments

- `$ARGUMENTS` - Project name (required). If not provided, ask the user.

## Phase 1: Fully Automated (run as one script)

All steps are deterministic for shadcn@4.0.2. Run in one bash invocation.

```bash
set -e

PROJECT_NAME="$ARGUMENTS"

# 1. Scaffold project (pinned version for deterministic output)
pnpm dlx shadcn@4.0.2 init -t vite -n "$PROJECT_NAME" --no-monorepo -b radix -d
cd "$PROJECT_NAME"

# 2. Install devtools
pnpm add -D oxlint oxfmt husky lint-staged knip

# 3. Initialize husky + configure pre-commit
pnpm exec husky init
echo 'pnpm lint-staged' > .husky/pre-commit

# 4. Remove Prettier (replaced by oxfmt)
pnpm remove prettier prettier-plugin-tailwindcss
rm -f .prettierrc .prettierignore

# 5. Fix ESLint: ignore shadcn generated UI components
#    shadcn@4.0.2 generates: globalIgnores(['dist'])
sed -i "s/globalIgnores(\['dist'\])/globalIgnores(['dist', 'src\/components\/ui'])/" eslint.config.js

# 6. Format everything with oxfmt
pnpm exec oxfmt .
```

## Phase 2: Configure package.json

Read `package.json`, then apply these changes (merge, don't overwrite — husky init may have already added `prepare`):

1. **Replace** the `scripts` section with:

```json
{
  "scripts": {
    "dev": "vite",
    "build": "tsc -b && vite build",
    "preview": "vite preview",
    "lint": "oxlint src/",
    "lint:eslint": "eslint .",
    "lint:all": "oxlint src/ && eslint .",
    "fmt": "oxfmt .",
    "fmt:check": "oxfmt --check .",
    "typecheck": "tsc --noEmit",
    "check": "oxlint src/ && oxfmt --check . && tsc --noEmit",
    "knip": "knip",
    "prepare": "husky"
  }
}
```

2. **Add** `lint-staged` config at top level:

```json
{
  "lint-staged": {
    "*.{js,jsx,ts,tsx}": ["oxlint --fix", "oxfmt"],
    "*.{json,css,md,yaml,yml,html}": ["oxfmt"]
  }
}
```

3. If pnpm showed "Ignored build scripts" warnings, add trusted packages to `pnpm-workspace.yaml`:

```yaml
onlyBuiltDependencies:
  - <package-name>
```

## Phase 3: Verify

```bash
pnpm fmt          # re-format after package.json edits
pnpm check        # oxlint + oxfmt --check + tsc --noEmit
pnpm build        # tsc -b && vite build
```

All three must pass. If anything fails, it likely means the shadcn template has changed — read the error, check the generated files, and adapt. Do NOT retry blindly.

## Phase 4: Initial commit

```bash
git add -A
git commit -m "Initial project setup: Vite + React + shadcn/ui + oxlint + oxfmt + husky"
```

```

├── .husky/pre-commit          # Git pre-commit hook -> lint-staged
├── src/
│   ├── components/
│   │   ├── ui/                # shadcn components (auto-generated, eslint-ignored)
│   │   └── theme-provider.tsx # Dark mode provider
│   ├── lib/utils.ts           # cn() utility
│   ├── App.tsx
│   ├── main.tsx
│   └── index.css              # Tailwind v4 + shadcn theme variables (oklch colors)
├── components.json            # shadcn CLI config
├── eslint.config.js           # ESLint 9 flat config (js/ts/tsx, react-hooks, react-refresh)
├── tsconfig.json              # Project references root (path aliases here)
├── tsconfig.app.json          # App source: ES2022, bundler mode, react-jsx, strict
├── tsconfig.node.json         # Node/Vite: ES2023, node types
├── vite.config.ts             # Vite + @tailwindcss/vite + @vitejs/plugin-react + path alias
└── package.json
```

## tsconfig Architecture

3-file project references pattern (official Vite scaffold, do NOT flatten):

- **tsconfig.json** — Root. `"files": []`, only `references` + `paths` alias (`@/*` -> `./src/*`).
- **tsconfig.app.json** — `src/` code. ES2022, `moduleResolution: "bundler"`, `verbatimModuleSyntax`, `react-jsx`, strict.
- **tsconfig.node.json** — `vite.config.ts`. ES2023, node types, same bundler mode.

> **Future note:** Vite 8 (currently beta) adds native `resolve.tsconfigPaths: true`, eliminating the duplicated path alias in `vite.config.ts`. When Vite 8 is stable, consider switching to `vite-tsconfig-paths` plugin or native support to define aliases in tsconfig only.

## Key Decisions

| Choice | Rationale |
|--------|-----------|
| `shadcn@4.0.2` (pinned) | Deterministic output, fully scriptable setup |
| oxlint + ESLint | oxlint for speed (50-100x), ESLint for type-aware + react-hooks rules |
| oxfmt over Prettier | 30x faster, broader format support (CSS/JSON/YAML/MD), adopted by major projects |
| Tailwind v4 | No config file, CSS-native, comes with shadcn |
| Husky + lint-staged | Industry standard pre-commit quality gates |
| knip | Dead code/unused deps detection (used by shadcn-admin, 11k+ stars) |
